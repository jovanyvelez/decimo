¿Por qué necesitamos el hexadecimal?

Imagina que eres diseñador y necesitas el rojo exacto de la camiseta de tu equipo. En digital, ese rojo es tres valores entre 0 y 255. El problema: 255 en binario es 11111111. Tres de esos, para cada color, en cada píxel de una imagen 4K: miles de millones de unos y ceros.

La solución que inventaron los ingenieros en los años 60: el hexadecimal. Con él, 11111111 binario se convierte en FF — dos caracteres en lugar de ocho.

El Sistema Decimal

El sistema decimal usa potencias de 10. Antes de aprender hexadecimal hay que dominar este puente: en cualquier sistema posicional, cada posición vale (base)^posición. Solo cambia la base.

Verificación (2×1000) + (7×100) + (4×10) + (8×1)
= 2000 + 700 + 40 + 8 = 2,748
Bloque central

El Sistema Hexadecimal

Base 16. Con los dígitos 0–9 solo tenemos 10 símbolos, pero necesitamos 16. La solución: incorporar las primeras seis letras del abecedario.

¡Atención! Las letras A–F son dígitos. 1F no es "uno F": es un número de dos dígitos cuyo segundo dígito vale 15.

Las Potencias de 16

Clave: 16² = 256 = 2⁸. Un byte puede almacenar valores de 0 a 255, que en hexadecimal son de 00 a FF. Por eso 1 byte = exactamente 2 dígitos hex. No es coincidencia: fue diseñado así.

Decimal → Hexadecimal

Regla de oro: Divide entre 16 (no entre 2 ni entre 8). Lee los residuos de abajo hacia arriba. Si el residuo es 10–15, escribe su letra: A B C D E F.
Dec → HexEjemplo 1 — Convertir 75 a hexadecimal
Dec → HexEjemplo 2 — Convertir 200 a hexadecimal
200 ÷ 16 = 12 residuo → 8 → 8 12 ÷ 16 = 0 residuo → 12 → C ↑ leer de aquí

Resultado: 200₁₀ = C8₁₆

C×16 + 8×1 = 12×16 + 8 = 192 + 8 = 200 ✓
Dec → HexEjemplo 3 — Convertir 3,500 (número grande)
3500 ÷ 16 = 218 residuo → 12 → C 218 ÷ 16 = 13 residuo → 10 → A 13 ÷ 16 = 0 residuo → 13 → D ↑ leer de aquí

Resultado: 3500₁₀ = DAC₁₆

D×256 + A×16 + C×1 = 13×256 + 10×16 + 12 = 3328 + 160 + 12 = 3500 ✓
Dec → HexEjemplo 4 — 255 y 65,535: los más importantes

255 → hex

255÷16=15 R 15→F 15÷16= 0 R 15→F → FF₁₆

65,535 → hex

65535÷16=4095 R F 4095÷16= 255 R F 255÷16= 15 R F 15÷16= 0 R F → FFFF₁₆
FF = máximo de 1 byte (8 bits). FFFF = máximo de 2 bytes (16 bits) = 65,535. Por eso el tipo unsigned short en C/C++ va de 0 a 65,535.
Conversión inversa · 40 min

Hexadecimal → Decimal

Regla de oro: Multiplica cada dígito hex por su potencia de 16 según su posición (derecha a izquierda desde 0). Recuerda: A=10, B=11, C=12, D=13, E=14, F=15. Luego suma.
Hex → DecEjemplo 5 — Convertir 2F₁₆ a decimal
Hex → DecA3₁₆
A×16¹ + 3×16⁰ = 10×16 + 3×1 = 160 + 3 = 163₁₀
Hex → Dec1B4₁₆
1×256 + B×16 + 4×1 = 256 + 11×16 + 4 = 256 + 176 + 4 = 436₁₀
Hex → DecEjemplo especial — DEAD₁₆ (clásico de programación)
D×16³ + E×16² + A×16¹ + D×16⁰ = 13×4096 + 14×256 + 10×16 + 13×1 = 53248 + 3584 + 160 + 13 = 57005₁₀
Nota curiosa: Los programadores usan DEAD, BEEF, CAFE, BABE como marcadores en memoria. Son fáciles de identificar en un volcado hexadecimal cuando algo falla.

Práctica guiada

G3: 3C₁₆ → decimal

3×16 + C×1 = 3×16 + 12 = 48 + 12 = 60₁₀ ✓

G4: B0₁₆ → decimal

B×16 + 0×1 = 11×16 + 0 = 176 + 0 = 176₁₀ ✓

G5: 7FF₁₆ → decimal

7×256 + F×16 + F×1 = 7×256 + 15×16 + 15 = 1792 + 240 + 15 = 2047₁₀ ✓

Conversiones Cruzadas Binario ↔ Hex

Esta es la razón más poderosa por la que el hex existe: convertir entre binario y hex es casi inmediato, sin divisiones.

El secreto: Cada dígito hexadecimal equivale exactamente a 4 bits (porque 2⁴ = 16). 1 dígito hex = 1 nibble = 4 bits. Nunca más, nunca menos.
Bin → HexEjemplo — Convertir 11010110₂ a hex
Verificación cruzada: D6₁₆ = 13×16+6 = 214₁₀ · en binario: 128+64+16+4+2 = 214₁₀ ✓
Hex → BinEjemplo — Convertir 5A₁₆ a binario
Hex → BinEjemplo clásico — 0xCAFE a binario
C → 1100 A → 1010 F → 1111 E → 1110 CAFE₁₆ = 1100 1010 1111 1110₂

Verificación: 12×4096 + 10×256 + 15×16 + 14 = 49152 + 2560 + 240 + 14 = 51966₁₀

Resumen y Cierre

Las 5 Reglas de Oro

1
Los 16 símbolos
0–9 son normales. A=10, B=11, C=12, D=13, E=14, F=15. Son dígitos, no letras. Sin ellos no hay hex.
2
Divisiones entre 16
Para Dec → Hex: divide entre 16, anota residuos, lee de abajo hacia arriba. Si el residuo es 10+, escribe la letra.
3
Potencias de posición
Para Hex → Dec: multiplica cada dígito por 16 elevado a su posición (derecha a izquierda desde 0) y suma.
4
Grupos de 4 bits
Para Bin ↔ Hex: 1 dígito hex = 4 bits exactos. Agrupa de 4 en 4 desde la derecha. Sin divisiones.
5
Verificación inversa
Siempre comprueba convirtiendo de vuelta. Si AB₁₆ = 171₁₀, entonces 171₁₀ debe dar AB₁₆.

Los 4 sistemas en perspectiva

SistemaBaseDígitosBits/dígito255 en este sistema
Binario20, 1111111111
Octal80–73377
Decimal100–9~3.32255
Hexadecimal160–F4FF

El hex gana en compacidad para datos binarios. 2 dígitos hex = 1 byte exacto. Por eso domina la programación de sistemas.

Glosario

Hexadecimal
Sistema numérico de base 16. Usa los dígitos 0–9 y las letras A–F.
Nibble
Grupo de 4 bits. Equivale exactamente a un dígito hexadecimal.
Byte
Grupo de 8 bits. Equivale a 2 dígitos hex. Puede valer de 0 a 255.
ASCII
Estándar que asigna un número decimal (y hex) a cada carácter del teclado.
RGB
Modelo de color: Rojo, Verde, Azul. Cada canal va de 0 a 255 (00 a FF).
Volcado de memoria
Representación de la RAM en formato hexadecimal, byte por byte.
Prefijo 0x
Notación de programación para indicar que un número está en hexadecimal.
Stop Code
Código de error del SO representado en hex. Ej: 0x0000007E.
Conversión cruzada
Convertir un valor entre dos o más sistemas para verificar su equivalencia.
